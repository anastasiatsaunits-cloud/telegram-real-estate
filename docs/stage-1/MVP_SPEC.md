# MVP Specification

## 1. Цель MVP

Сделать первую рабочую версию продукта внутри Telegram, которая позволяет:
- завести пользователя через бота,
- открыть Mini App,
- провести через короткий квиз,
- показать подборку объектов,
- открыть карточку объекта,
- захватить телефон,
- передать лид в CRM.

MVP не должен быть перегружен. Его задача: быстро запуститься, показать ценность пользователю и начать собирать реальные лиды.

---

## 2. Что входит в MVP

### 2.1 Telegram Bot
Входит:
- команда `/start`
- welcome message
- кнопка открытия mini app
- базовый возврат пользователя в каталог
- отправка сервисного сообщения после заявки

Не входит пока:
- сложные реактивации
- автоматические воронки прогрева
- массовые рассылки
- сложные сценарии follow-up

---

### 2.2 Mini App
Входит:
- главная витрина
- квиз из 3 шагов
- экран `Подборка готова`
- форма телефона
- список объектов
- карточка объекта

Не входит пока:
- личный кабинет
- избранное
- сравнение объектов
- мультиязычность
- сложный recommendation engine

---

### 2.3 Backend
Входит:
- auth/session для Telegram Mini App
- каталог объектов
- получение регионов
- сохранение ответов квиза
- базовый подбор объектов
- создание лида
- отправка лида в CRM
- базовая аналитика событий

Не входит пока:
- сложная сегментация
- scoring lead quality
- сложные очереди маркетинга
- расширенные роли пользователей

---

### 2.4 CRM
Входит:
- создание лида
- передача телефона
- передача Telegram ID
- передача параметров квиза
- передача просмотренного объекта или подборки

Не входит пока:
- двусторонняя синхронизация
- автоматизация стадий CRM
- сложные роботы

---

## 3. Пользовательский поток MVP

1. Пользователь открывает Telegram-бот
2. Нажимает `Открыть каталог`
3. Открывается Mini App
4. Видит главную витрину
5. Нажимает `Подобрать объект`
6. Проходит 3 шага квиза:
   - регион
   - бюджет
   - срок покупки
7. Видит экран `Подборка готова`
8. Оставляет телефон
9. Получает подборку объектов
10. Открывает карточку объекта
11. Отправляет заявку / запрос консультации
12. Лид уходит в CRM

## 5. Сущности MVP

### Пользователь
- Telegram ID
- имя
- username
- телефон

### Квиз
- регион
- бюджет
- срок покупки

### Объект
- название
- регион
- локация
- цена
- описание
- фото
- базовые инвест-показатели

### Лид
- контакт
- ответы квиза
- источник
- просмотренный объект

## 6. API MVP

### Bot / Session
- `POST /session/webapp-auth`
- `GET /session/me`

### Catalog
- `GET /regions`
- `GET /properties`
- `GET /properties/:id`

### Quiz
- `POST /quiz/answer`
- `POST /quiz/complete`
- `GET /recommendations`

### Leads
- `POST /leads`
- `POST /applications`

### Analytics
- `POST /events`

- bot_started
- miniapp_opened
- quiz_started
- quiz_step_completed
- quiz_completed
- phone_form_opened
- phone_submitted
- catalog_opened
- property_opened
- application_submitted

---

## 8. Критерии готовности MVP

MVP считается готовым, если:
- бот запускается
- Mini App открывается внутри Telegram
- квиз работает от начала до конца
- подборка объектов отображается
- карточка объекта открывается
- телефон сохраняется
- лид создаётся в CRM
- ключевые события логируются
