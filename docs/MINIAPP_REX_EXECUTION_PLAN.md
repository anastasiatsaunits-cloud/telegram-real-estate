# Задачи для исполнения

## Блок A. Home / первый экран

### A1. Сделать home screen визуально сильнее и ближе к референсу
Что нужно:
- приветственный premium hero
- market chooser как главный action-layer
- приоритетные рынки крупнее второстепенных
- визуальный акцент на Крым и Сочи
- ощущение закрытого каталога / private access

Критерий приёмки:
- home не выглядит как демо-список ссылок
- home визуально продаёт переход в каталог

### A2. Bottom CTA cluster
Что нужно:
- довести нижние CTA до консистентного UI
- визуально они должны быть частью funnel, а не случайными кнопками

Критерий приёмки:
- три CTA выглядят единым блоком

---

## Блок B. Список объектов

### B1. Привести property cards к premium mobile style
Что нужно:
- сильный hero block
- крупная цена
- чёткая типографика
- region / city / positioning
- понятный CTA
- хорошие радиусы, воздух, иерархия

Критерий приёмки:
- карточки выглядят как продукт premium сегмента, а не список из CRM

### B2. Уточнить сортировку и рынок
Что нужно:
- убедиться, что market chooser реально открывает нужный рынок
- Крым → только Крым
- Сочи → только Сочи

Критерий приёмки:
- регионы не смешиваются

---

## Блок C. Detail page

### C1. Доработать hero section detail page
Что нужно:
- крупный hero block
- место под gallery / image pagination style
- цена как отдельный premium блок
- сильный заголовок + гео

### C2. Сделать структуру detail page ближе к референсу
Что нужно:
- блок "о проекте"
- блок быстрых фактов
- блок вариантов покупки
- CTA "инвест расчёт"
- CTA "оставить заявку"

### C3. Добавить ощущение product story, а не data dump
Что нужно:
- хороший вертикальный ритм
- меньше сухих строк "поле: значение"
- больше curated layout

Критерий приёмки:
- карточка воспринимается как продающий экран объекта

---

## Блок D. UI foundation / motion

### D1. Использовать UI primitives
Что нужно:
- не плодить новые случайные inline patterns
- опираться на `components/ui.tsx`
- при необходимости расширить его

### D2. Добавить первые motion-эффекты
Что нужно:
- мягкое появление market cards
- мягкое появление property cards
- без перегруза, без дешёвых эффектов

Критерий приёмки:
- интерфейс ощущается живым, но сдержанным

---

## Блок E. Техническая чистота

### E1. Не ломать сборку
Обязательно:
- `pnpm --filter miniapp build`
- без новых build warnings, если можно убрать быстро

### E2. Не ломать live endpoints
Обязательно проверить после правок:
- `/api/properties`
- `/api/regions-proxy`
- `/`
- `/properties`
- detail page любого объекта

---

# Формат исполнения

1. Делать блоками, не хаотично
2. После каждого заметного блока — короткий список что изменено
3. Если упираешься в спорный UX-выбор, предлагай 2 коротких варианта, не больше
4. Не переписывать продукт заново, а усиливать уже рабочий слой
