# TG warmup → miniapp reference flow

Источник: скрины Анастасии от 19.04.2026.

## Что фиксируем этим референсом

Это не просто визуал, а логика пользовательского пути:

1. Прогрев в Telegram-боте до открытия каталога
2. Сообщение с коротким позиционированием и обещанием ценности
3. Кнопка / CTA на открытие каталога
4. Первый экран miniapp с крупными market cards:
   - Крым
   - Сочи
   - Таиланд
   - Горный кластер
5. Нижние CTA-кнопки:
   - о компании
   - отзывы
   - заработать
6. Листинг объектов в формате premium mobile cards
7. Детальная карточка объекта:
   - крупное фото / gallery
   - заголовок
   - гео
   - цена от
   - блок "о проекте"
   - компактные факты (площадь, цель)
   - варианты покупки
   - CTA "инвест расчет"
   - CTA "оставить заявку"

## Практический вывод для miniapp

Витрина должна строиться не как сухой каталог JSON, а как mobile-first premium funnel:
- сначала выбор рынка
- потом curated карточки
- потом detail page с заявкой
