## 2. Что уже обнаружено как обязательное

### 2.1 Нужно установить `pnpm`
Почему:
- проект делаем как monorepo,
- `pnpm` удобнее и чище для workspace-структуры,
- проще управлять `apps/bot`, `apps/miniapp`, `apps/api`, `packages/*`.

Команда установки:
```bash
npm install -g pnpm
```

Проверка:
```bash
pnpm -v
```

## 3. Что желательно установить сразу для старта разработки

### 3.1 Docker Desktop
Нужен для:
- PostgreSQL
- Redis
- локального окружения backend
- одинаковой среды разработки

Проверка:
```bash
docker --version
```

Если Docker не установлен, это один из первых обязательных шагов.

### 3.2 TablePlus / DBeaver / любой GUI-клиент к PostgreSQL
Нужен для:
- просмотра таблиц,
- проверки данных,
- ручной валидации схемы.

### 3.3 Figma
Нужна для первого этапа, если делаем нормальный discovery + экраны + UX flow.

### 3.4 Postman или Bruno
Нужен для:
- проверки API,
- тестирования backend endpoints,
- фиксации коллекций запросов.

Рекомендация: Bruno, потому что проектовый и легче хранится в git.
