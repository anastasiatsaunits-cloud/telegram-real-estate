# Product roadmap — telegram-real-estate

Обновлено: 19.04.2026

## Цель

Сделать miniapp уровня premium product:
- сильная TG → miniapp воронка
- стабильный деплой и проверки
- красивая mobile-first витрина
- качественная карточка объекта
- нормальные рынки и фильтры
- потом аналитика, умный поиск и визуальное усиление

---

# Первая очередь — сделать как обязательную оснастку и довести до первого сильного рабочего прототипа

## 1. Правильная модель рынков и регионов
Обязательно сделать в первой очереди.

Что входит:
- развести Сочи и Крым по отдельным регионам/рынкам
- убрать ситуацию, когда объекты Крыма лежат в `region_sochi`
- проверить выдачу регионов в backend и miniapp
- подготовить основу под следующие рынки: Таиланд, горный кластер

Результат:
- честный market chooser
- корректные фильтры
- правильная структура каталога

## 2. Premium UX/UI витрины
Обязательно сделать в первой очереди.

Что входит:
- home screen по логике TG warmup
- market chooser первого экрана
- premium mobile cards в списке объектов
- premium detail page с сильной иерархией
- дорогая тёмная/нейтральная визуальная система

Результат:
- miniapp уже можно показывать как продукт, а не как техдемо

## 3. DevOps toolkit / setup control
Обязательно сделать в первой очереди.

Что входит:
- `scripts/setup-check.sh`
- `scripts/smoke-check.sh`
- `scripts/deploy-api.sh`
- `scripts/deploy-miniapp.sh`
- predeploy checklist
- postdeploy smoke check

Результат:
- Мия не спотыкается о монорепу и деплой по кругу
- проект становится воспроизводимым

## 4. Design system / UI foundation
Обязательно сделать в первой очереди.

Что входит:
- единые кнопки
- единые карточки
- единые отступы / радиусы / типографика
- единая палитра
- подготовка к переходу на более системный UI-слой

Результат:
- интерфейс не распадается на случайные блоки
- проще масштабировать новые экраны

## 5. Framer Motion / микроанимации
Обязательно сделать в первой очереди.

Что входит:
- мягкие появления market cards
- переходы list → detail
- живые CTA
- аккуратная анимация без перегруза

Результат:
- продукт ощущается «дороже» и живее

## 6. GitHub Actions + deploy runbook + Vercel/Render control
Обязательно сделать в первой очереди.

Что входит:
- CI pipeline для build miniapp/api
- smoke checks после изменений
- runbook по Render/Vercel
- rollback path
- cache clear / redeploy notes

Результат:
- инфраструктура перестаёт держаться на ручных импровизациях
- Мия может быстрее и увереннее вносить изменения

# Текущее правило исполнения

Сейчас первоочередная обязательная очередь — **пункты 1-6**.

Пока они не доведены до сильного рабочего прототипа:
- не распыляться на тяжёлые R&D-фичи
- не уходить в 3D и semantic search раньше времени

Когда первый сильный рабочий прототип готов:
- брать вторую очередь, пункты **7-10**
- докручивать продукт до экстра-класса
